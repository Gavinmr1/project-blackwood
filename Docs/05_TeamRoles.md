# Team Roles

Version 1.0

---

# Purpose

Project Blackwood is developed by a small team.

To keep development organized, every developer has clear areas of ownership.

This does **not** prevent collaboration.

Instead, it helps avoid duplicate work and conflicting implementations.

---

# Lead Developer

Primary Responsibilities

- Gameplay programming
- C++ systems
- Blueprint architecture
- Unreal Engine systems
- Core gameplay
- Multiplayer architecture
- Save system
- Inventory
- Equipment
- Tasks
- AI systems
- Performance
- Project architecture

The Lead Developer is responsible for maintaining the overall technical direction of Project Blackwood.

Architectural changes should be reviewed before implementation.

---

# World Developer

Primary Responsibilities

- Environment art
- Landscape creation
- Level dressing
- Props
- Foliage
- Lighting
- Audio placement
- Item Data Assets
- Wildlife data
- Journal entries
- Ranger notes
- Campgrounds
- Trail layouts

The World Developer should avoid modifying core gameplay systems unless coordinating with the Lead Developer.

---

# Shared Responsibilities

Both developers should:

- Test gameplay
- Report bugs
- Improve documentation
- Suggest gameplay improvements
- Review new systems
- Maintain project organization

---

# AI Usage

Each developer may use AI assistance.

However:

AI should extend existing systems rather than replacing them.

AI should never redesign architecture without discussion.

If AI recommendations conflict with existing documentation, follow the documentation.

---

# Branch Strategy

Each developer should primarily work on independent features.

Large architectural changes should be completed on separate branches before merging into main.

---

# Goal

Every developer should know exactly:

What they own.

What they should avoid modifying.

Who to ask before changing architecture.

Keeping responsibilities clear allows multiple developers to work efficiently without creating conflicts.
