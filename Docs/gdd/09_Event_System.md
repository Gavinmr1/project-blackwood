# Event System

## Overview

The Event System controls the dynamic events that occur throughout the park.

Events create unexpected situations that make each play session unique.

Events range from normal park problems to unexplained supernatural encounters.

---

# Event Philosophy

Events should feel like natural occurrences.

Players should discover events through gameplay.

The system should avoid making events feel scripted.

---

# Event Categories

## Environmental Events

Natural changes in the park.

Examples:
- Storms
- Heavy rain
- Flooding
- Snow
- Fallen trees
- Landslides

Effects:
- Change traversal
- Create tasks
- Alter wildlife behavior

---

# Emergency Events

Situations requiring ranger response.

Examples:
- Lost hikers
- Injured visitors
- Vehicle accidents
- Missing campers
- Medical emergencies

Effects:
- Encourage teamwork
- Create urgency
- Reward rescue skills

---

# Wildlife Events

Interactions with animals.

Examples:
- Bear encounters
- Injured animals
- Migration
- Predator activity
- Unusual behavior

Effects:
- Encourage tracking
- Teach ecosystem behavior

---

# Maintenance Events

Park infrastructure problems.

Examples:
- Broken bridges
- Power failures
- Damaged signs
- Equipment failures

Effects:
- Give players meaningful work

---

# Mystery Events

Unexplained occurrences.

Examples:
- Strange lights
- Missing equipment
- Unknown tracks
- Radio interference
- Unusual sounds

Effects:
- Create curiosity

---

# Cryptid Events

Rare supernatural encounters.

Examples:
- Bigfoot sightings
- Unknown creatures
- Strange phenomena
- Abductions
- Paranormal activity

These events should be rare.

---

# Event States

The park has an invisible activity level.

Possible states:

## Calm

Normal park operations.

---

## Uneasy

Small unusual signs appear.

Examples:
- Missing wildlife
- Strange noises
- Visitor reports

---

## Active

Strange events become more noticeable.

Examples:
- Evidence appears
- Equipment fails
- Wildlife reacts

---

## Hostile

Dangerous situations occur.

Examples:
- Rescue emergencies
- Dangerous creatures
- Severe hazards

---

# Event Selection

The system considers:
- Time of day
- Weather
- Season
- Location
- Player progress
- Previous events
- Park condition

---

# Event Escalation

Events should build slowly.

Example:

Day 1:
Strange footprints.

Day 10:
Animals avoiding an area.

Day 20:
Equipment disappears.

Day 30:
A player sees something watching them.

---

# Player Choice

Players should influence events.

Examples:

Ignoring problems:
- Park condition decreases
- Events become worse

Helping the park:
- Improves stability

---

# Design Rule

Events should create stories, not objectives.

Players should say:
"I can't believe that happened."

Not:
"We completed another event."
